## [cite_start]1. Descripción del Proyecto [cite: 6]

[cite_start]Este proyecto tiene como objetivo diseñar e implementar una aplicación en Python que cubra el flujo completo de trabajo de un análisis de ciencia de datos, empleando la Programación Orientada a Objetos (POO) como paradigma central de desarrollo[cite: 7]. [cite_start]El sistema modela las principales etapas de un pipeline de ciencia de datos: desde la carga y gestión de datos hasta la evaluación de modelos predictivos y la visualización de resultados[cite: 8]. 

[cite_start]El diseño prioriza la extensibilidad, la separación de responsabilidades y la reutilización de componentes mediante el uso de clases abstractas, herencia y composición[cite: 9]. [cite_start]La arquitectura está organizada en módulos independientes que se comunican a través de interfaces bien definidas, permitiendo sustituir o extender cualquier componente sin afectar al resto del sistema[cite: 10]. [cite_start]Este enfoque modular facilita el mantenimiento y la escalabilidad del proyecto a lo largo de sus diferentes fases de desarrollo[cite: 11].

### [cite_start]Objetivos Específicos [cite: 12]

* [cite_start]Aplicar los principios de la POO (abstracción, encapsulamiento, herencia y polimorfismo) en cada módulo del sistema[cite: 13].
* [cite_start]Diseñar una jerarquía de clases clara y extensible para los modelos de aprendizaje automático[cite: 14].
* [cite_start]Establecer una arquitectura modular que permita el desarrollo incremental del sistema[cite: 15].
* [cite_start]Documentar el diseño mediante diagramas UML antes de proceder a la implementación[cite: 16].
